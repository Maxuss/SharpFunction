Execute command has a very complex syntax tree
Execute parameters

  1. ExecuteCondition.Align
    Updates the command's execution position, aligning to its current block position.
    Syntax:
    extraParameters [0] should have swizzle of coordinates.
    Example: "xzy" or "yz"
  1. ExecuteCondition.Anchored
    Sets the execution anchor to the eyes or feet. Defaults to feet.
    Syntax:
    extraParameters [0] should be AnchorCondition
  1. ExecuteCondition.As
    Sets the command's executor to target entity, without changing execution position, rotation, dimension, or anchor
    Syntax:
    extraParameters [0] should be EntitySelector
  1. ExecuteCondition.At
    Sets the execution position, rotation, and dimension to match those of an entity; does not change executor.
    Syntax:
    extraParameters [0] should be EntitySelector
  1. ExecuteCondition.Facing
    Sets the execution rotation to face a given point, as viewed from its anchor (either the eyes or the feet)
    Syntax:
    extraParameters [0] should be FacingCondition
    First option (FacingCondition.Position):
    extraParameters [1] should be Vector3)
    Second option (FacingCondition.Entity):
    extraParameters [1] should be EntitySelector entity
    extraParameters [2] should be AnchorCondition anchor
  1. ExecuteCondition.In
    Sets the execution dimension and position
    Syntax:
    extraParameters [0] should be dimension string ("overworld"|"the_nether"|"the_end"|"custom dimension")
  1. ExecuteCondition.Positioned
    Sets the execution position, without changing execution rotation or dimension;
    Syntax:
    extraParameters [0] should be Vector3
  1. ExecuteCondition.PositionedAs
    Works like ExecuteCondition.Positioned but anchors at entity
    Syntax:
    extraParameters [0] should be EntitySelector
  1. ExecuteCondition.Rotated
    Sets the execution rotation
    Syntax:
    extraParameters [0] should be yaw float
    extraParameters [1] should be pitch float
  1. ExecuteCondition.RotatedAs
    Sets the execution rotation to entity's rotation
    Syntax:
    extraParameters [0] should be EntitySelector entity

Execute Conditions (if/unless)

  1. ExecuteSubcondition.Block
    Compares the block at a given position to a given block ID or block tag.
    Syntax:
    extraParameters [0] should be Vector3
    extraParameters [1] should be string block predicate tag in format of namespaced_id[predicate_states]{nbt_tags}
  1. ExecuteSubcondition.Blocks
    Compares the blocks in two equally sized volumes
    Syntax:
    extraParameters [0] should be Vector3 array, where [0] is beginning of volume, [1] is end of volume and [2] is destination of volume
    extraParameters [1] should be ScanningMode
  1. ExecuteSubcondition.NBTData
    Checks whether the targeted block, entity or storage has any data tag for a given path
    Syntax:
    extraParameters [0] should be NBTDataType and depending on it should be extraParameters [1]
    extraParameters [2] should always be nbt data string to check for.
    3.1. NBTDataType.Block
    extraParameters [1] should be Vector3 block pos to check
    3.2. NBTDataType.Entity
    extraParameters [1] should be EntitySelector entity to check
    3.3. NBTDataType.Storage
    extraParameters [1] should be string namespaced id to check
  1. ExecuteSubcondition.Entity
    Checks whether one or more entities exist.
    Syntax:
    extraParameters [0] should be EntitySelector entity to check for.
  1. ExecuteSubcondition.Predicate
    Checks whether the predicate evaluates to a positive result.
    Syntax:
    extraParameters [0] should be namespaced id for predicate to seek.
  1. ExecuteSubcondition.Scoreboard
    Check whether a score has the specific relation to another score, or whether it is in a given range.
    Syntax:
    extraParameters [0] should be ScoreComparation and other params will be dependant on it's value.

6.1. ScoreComparation.Comparation
Other params should go this way:
EntitySelector holder of the objective to be compared.
string name of the objective to be compared the value.
Comparator comparator to compare the value.
EntitySelector holder of the objective to compare.
string name of the objective to compare the value.

6.2. ScoreComparation.Match
Other params should go this way:
EntitySelector holder of the objective to be matched.
string name of the objective to be matched the value.

Execute Store (result|success)

  1. ExecuteStore.Block
    Saves the final command's return value as tag data within a block entity. Store as a byte, short, int, long, float, or double.
    Syntax:
    extraParameters [0] should be Vector3 of block
    extraParameters [1] should be NBTPath to store data
    extraParameters [2] should be Type of data to be stored
    extraParameters [3] should be double scale for value to be rounded if it is decimal
  1. ExecuteStore.Bossbar
    Saves the final command's return value in either a bossbar's current value or its maximum value
    Syntax:
    extraParameters [0] should be string namespaced id of bossbar to store data
    extraParameters [1] should be BossbarOverwrite of value to overwrite (current or max)
  1. ExecuteStore.Entity
    Save the final command's return value in one of an entity's data tags. Store as a byte, short, int, long, float, or double. Like the /data command, /execute store entity cannot modify player's NBT.
    Syntax:
    extraParameters [0] should be EntitySelector of entity
    extraParameters [1] should be NBTPath to store data
    extraParameters [2] should be Type of data to be stored
    extraParameters [3] should be double scale for value to be rounded if it is decimal
  1. ExecuteStore.Score
    Overrides the score held by targets on the given objective with the final command's return value.
    Syntax:
    extraParameters [0] should be EntitySelector of score holder
    extraParameters [1] should be string namespaced id of scoreboard
  1. ExecuteStore.Storage
    Uses the path within storage target to store the return value in. Store as a byte, short, int, long, float, or double. If the storage does not yet exist, it gets created.
    Syntax:
    extraParameters [0] should be string namespaced id of storage to store data
    extraParameters [1] should be NBTPath to store data
    extraParameters [2] should be Type of data to be stored
    extraParameters [3] should be double scale for value to be rounded if it is decimal
